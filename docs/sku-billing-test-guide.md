# SKU 参数计费 — 手动测试指南（小白版）

> 这份文档教你一步步测试「按请求参数额外计费」的新功能。
> 不需要懂代码，跟着点就行。看到 `[ ]` 就是一个检查项，做完打个勾 ✅。

---

## 这个功能是干嘛的？（先看懂再测）

一句话：**让你能给"大图、高清、长视频"这种请求多收钱。**

举个例子你就懂了：

- 一张普通图 → 收 1 倍价钱
- 一张超大图（4K）→ 你可以设成收 4 倍价钱
- 一个高清画质（hd）→ 你可以设成收 2 倍价钱

以前这些都写死在代码里改不了，现在你能在后台自己配，像填表格一样。

**几个关键词**（看一眼混个眼熟）：

| 词 | 大白话 |
|----|--------|
| 倍率 | 就是"乘几倍价钱"，比如 2 就是双倍 |
| 规则 | 一条"什么情况收几倍"的设定 |
| 模型通配 | 这条规则管哪些模型，比如 `dall-e-3` 或 `sora*`（星号=开头是 sora 的都算） |
| 取值参数键 | 看请求里的哪个字段，比如 `size`（尺寸）、`quality`（画质） |
| 输出键 | 这倍率在账单上叫什么名字，建议用 `sku_` 开头方便认 |

## 第一部分：配置页能不能用（最简单，先测这个）

### 怎么进入配置页

1. 登录后，点左边菜单的 **系统设置**
2. 找到 **计费设置**（Billing）
3. 找到 **Model Pricing**（模型定价）这张卡片
4. 卡片上方有几个小标签，点最后那个 **参数计费**（Parameter-based pricing）

> 💡 嫌麻烦可以直接复制这个网址进去：
> `http://localhost:3000/console/system-settings/billing/model-pricing`

### 检查清单

- [x] **1-1** 能看到「参数计费」这个标签
- [x] **1-2** 刚进去时：开关是**关着**的、上限填的是 **0**、规则列表是**空的**（写着"暂无规则"）
- [x] **1-3** 点「添加规则」，蹦出来一张卡片，里面有：模型通配、取值参数键、类型、输出键这些输入框
- [ ] **1-4** 把「类型」下拉框从"数值分档"切到"枚举映射"再切到"有无加价"，下面那个填详情的框，提示文字和默认内容会跟着变
- [ ] **1-5** 删掉刚才加的规则（点垃圾桶图标），规则又变空了

## 第二部分：快速灌入一批示例规则（省得你一条条填）

### 操作步骤

1. 在「参数计费」页面，点右上角 **Switch to JSON**（切到 JSON 模式）
2. 把下面这段**整个复制**，粘贴进那个大文本框（覆盖原来的内容）：

```json
[
  {"models":["dall-e-3"],"source":"quality","kind":"enum","out_key":"sku_quality","enabled":true,"enum":{"standard":1,"hd":2}},
  {"models":["gpt-image-1"],"source":"size","kind":"tier","out_key":"sku_size","enabled":true,"derive":"long_edge","tiers":[{"up_to":1024,"ratio":1,"label":"1K"},{"up_to":2048,"ratio":2,"label":"2K"},{"up_to":0,"ratio":4,"label":"4K+"}]},
  {"models":["sora*"],"source":"size","kind":"tier","out_key":"size","enabled":true,"derive":"long_edge","tiers":[{"up_to":1280,"ratio":1},{"up_to":0,"ratio":2}]},
  {"models":["sora*"],"source":"duration","kind":"tier","out_key":"sku_duration","enabled":true,"derive":"number","tiers":[{"up_to":5,"ratio":1},{"up_to":0,"ratio":1.8}]}
]
```

3. 点 **Switch to Visual**（切回可视化），应该能看到刚粘的 4 条规则变成了卡片
4. 把最上面的 **「启用 SKU 倍率表」开关打开**
5. 点最下面的 **保存**

### 检查清单

- [ ] **2-1** 粘贴 JSON 后切回可视化，能看到 4 条规则卡片
- [ ] **2-2** 保存后弹出绿色提示「SKU 倍率设置已保存」

> 这 4 条规则的意思（翻译给你听）：
> - dall-e-3 的画质：标准 ×1，高清 ×2
> - gpt-image-1 的尺寸：1K 以内 ×1，2K ×2，更大 ×4
> - sora 视频尺寸：1280 以内 ×1，更大 ×2
> - sora 视频时长：5 秒以内 ×1，更长 ×1.8

## 第三部分：防呆校验（故意填错，看它拦不拦）

这部分是验证"系统会不会阻止你配出会算错钱的规则"。

### 检查清单

- [ ] **3-1 禁用危险输出键**：随便加一条规则，把「输出键」填成 `seconds`，保存 → **应该报错被拦住**（提示输出键不能是 n / seconds / duration）
  > 为什么？因为 `seconds` 是系统用来记"几秒"的，你占用它会把计费搞乱。
- [ ] **3-2 必填检查**：加一条规则，「取值参数键」留空，保存 → **应该提示要填**
- [ ] **3-3 模型必填**：「模型通配」留空，保存 → **应该提示至少填一个模型**
- [ ] **3-4 JSON 写错**：详情框里乱填（比如删掉一个括号），框下面 → **应该出现红色报错文字**，且保存按钮不让你存

## 附：常见疑问

**Q：我配了规则但好像没生效？**
A：先确认最上面那个「启用 SKU 倍率表」开关是**打开**的，并且**点了保存**。开关关着时整个功能不工作（这是故意的，默认关闭最安全）。

**Q：上限（Max total multiplier）是干嘛的？**
A：防止你不小心配太狠。比如设成 3，那不管多少条规则叠加，最多收 3 倍封顶。填 0 = 不封顶。

**Q：会不会影响我现有的其它计费？**
A：不会。这个功能默认关闭，只做"额外叠加"，不动任何原有价格。关掉开关就跟没这功能一模一样。
